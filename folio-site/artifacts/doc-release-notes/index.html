Folio v0.2 Release Notes

Folio v0.2 makes static browser artifacts stricter and more useful: seven named artifact kinds, a static-only ledger, a visual baseline, and an auth probe before any external host publish.

Summary

The skill now treats every deliverable as a self-contained artifact with one entry HTML file, an honest manifest ledger, and binary gates before handoff. The default remains local-first: build a folder that opens in a browser, then publish only when the user explicitly asks.

The release also narrows scope. Folio owns browser-deliverable docs, sheets, dashboards, explainers, slides, tools, and landing pages; backend APIs, databases, authenticated apps, and production SPA shells remain out of scope.

New Artifact Kinds

The v0.2 kind list replaces broad v0.1 names with reader-centered shapes. Old stamps still map forward when an artifact is audited or remixed.

v0.1 kindv0.2 kindReason
reportdocReadable structured prose with tables and code.
visualizationexplainer or dashboardUse dashboard for metrics, explainer for a data story.
demoexplainerWalk a viewer through a concept or artifact flow.
prototypetoolOne task accomplished with local state and inline data.
playgroundtoolInteractive controls without accounts or persistence.
dashboarddashboardMetrics-focused shape is unchanged.
landinglandingOrientation and navigation are unchanged.

Static-Only Discipline

Artifacts do not call live services. v0.2 explicitly forbids browser network reads, socket streams, and live XHR; data is embedded inline, loaded from a local file, or generated at build time.

The manifest ledger has no api kind in this release. Data, mock data, images, scripts, styles, fonts, and publish targets are ledger rows; secrets never belong in the artifact source.

Visual Baseline

Folio now enforces a small but strict design floor: OKLCH tokens, system fonts, readable measure, no italic headings, no purple-to-blue washes, no card-in-card layouts, and overflow-x: clip on the page.

:root {
  fonts: display / body / mono;
  palette: paper, ink, rule, accent, status;
  scale: space tokens, radius tokens, shadows;
}
[data-theme='dark'] {
  palette: same token names with dark values;
}

Auth Probe

Publish is still optional, but v0.2 documents host profiles for Surge, GitHub Pages, Cloudflare Pages, and Cloudflare Workers. Before any deploy, the agent silently runs the matching auth probe; if it fails, the user gets one clear login instruction and no fabricated URL.

What's Next

Roadmap items include worked examples, an optional verification script, machine-readable manifest handoff, and opt-in API integration in v0.3 with offline degradation. The current release keeps the main promise narrow: static artifacts that open locally and can be handed off honestly.

Gates for this sample: opens-locally, console-clean, ledger-complete, no-secrets, mock-labeled, static-only, and visual-baseline.
