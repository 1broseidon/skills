The Folio Loop

Every Folio run turns a request into a static artifact through seven named steps: Inventory → Picks → Ledger → Build → Verify → Gates → Handoff.

01

Inventory

Read the request, inputs, existing artifact tree, and conventions before writing HTML. Evidence is labeled observed, derived, stated, or absent so the artifact does not pretend unknown data is real.

Evidence
observedderivedstatedabsent
02

Picks

Choose one artifact kind, a bundle type, host profile, and audience. In v0.2 the valid kinds are doc, sheet, dashboard, explainer, slides, tool, and landing.

kind=explainer
bundle=single-file
host=local
audience=human demo
03

Ledger

The manifest ledger records every dependency, data file, mock, image, font, and publish target before build. External API calls are not ledgerable in v0.2 because static-only is a gate.

Mini ledger
depevidenceinline dataobservedsurge URLdeferred
04

Build

Build the smallest browser artifact that satisfies the request. For single-file bundles, CSS and JS live inside index.html; visual values resolve through tokens and system fonts.

Runnable first, polish second.

05

Verify

Open the artifact locally and exercise the primary interaction. If local files are fetched, use a static server; if the bundle is inline-only, it should also work from file://.

✓ entry exists
✓ interaction works
✓ console clean
06

Gates

Apply binary checks: opens-locally, console-clean, ledger-complete, no-secrets, mock-labeled, static-only, and visual-baseline. Any failed gate triggers a fix loop before handoff.

✓ static-only
✓ visual-baseline
✓ no-secrets
07

Handoff

Report the path, how to open it, ledger summary, verification results, and any deferred scope. Publish URLs are only named when copied from real host command output.

pathopenledgerchecks
